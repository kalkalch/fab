# Требования: VK-бот для FAB

## Общие требования (принципы разработки)

- Делать всё согласно официальной документации (API, библиотеки, платформы).
- При необходимости читать официальную документацию, не полагаться на устаревшие примеры или предположения.
- Ничего не придумывать: не добавлять недокументированное поведение, не изобретать свои форматы или контракты без явной необходимости.
- Учитывать, что приложение работает в Docker: пути к файлам, порты, переменные окружения, сеть и ограничения контейнера должны быть совместимы с докером.

---

## Цели

- Добавить бота для ВКонтакте с функционалом, аналогичным Telegram-боту.
- Доступ к боту только для пользователей из списка (отдельный список для VK, связывать пользователей VK и Telegram не нужно).
- Приложение не должно падать при отсутствии связи с VК.
- В логах должно быть видно, откуда пришёл пользователь (Telegram или VK).

---

## Функционал VK-бота (аналогично Telegram)

- Команды: старт, помощь.
- Кнопки: добавить доступ (ссылка на веб), мои доступы, помощь, главное меню, смена языка.
- Для админов: управление пользователями — добавить в список, список пользователей, удалить из списка.
- Текстовые сообщения: добавление пользователя в белый список по ID (для админов).

---

## Авторизация

- Админы VK: из конфига (например, `ADMIN_VK_IDS`).
- Остальные: белый список в БД (отдельный «контур» для VK, без связи с Telegram).
- Проверка: при каждом действии проверять, что пользователь админ или в белом списке для своей платформы.

---

## База данных (минимальные изменения)

- Один новый столбец во всех затронутых таблицах: **`source`** (`'telegram'` | `'vk'`).
- Существующие столбцы `telegram_user_id` и `chat_id` используются и для VK: хранят VK user id и peer_id.
- **whitelist_users**: добавить `source`, уникальность по паре `(source, telegram_user_id)`.
- **user_sessions**: добавить `source`.
- **access_requests**: добавить `source`.
- Индексы по `(source, telegram_user_id)` для выборок по пользователю.
- Миграция: для существующих строк `source = 'telegram'` (DEFAULT при добавлении столбца).
- Единый контракт в коде: везде передавать `(source, user_id, peer_id)`; один слой доступа в БД с явным `source` в запросах.

---

## Устойчивость

- Запуск VK-бота в отдельной задаче/потоке.
- При ошибке подключения к VK: только логирование (warning/error), без исключения в главный цикл — приложение продолжает работать (Telegram, веб, MQTT и т.д.).
- Опционально: периодическая попытка переподключения VK-бота.

---

## Логирование

- При каждом действии пользователя в логах указывать источник: `source=telegram` или `source=vk` (или явная пометка в тексте лога, например «User 123 (vk) …»).

---

## Что не делаем

- Связывать пользователей VK и Telegram (разные списки, разные ID).
- Падать при недоступности VK API.
- Дублировать таблицы или пары столбцов (vk_user_id рядом с telegram_user_id) — только один столбец `source` и те же числовые поля.

## Переменные окружения

Все переменные, которые использует приложение (текущие и планируемые для VK).

### Обязательные (без них приложение не стартует)

| Переменная | Описание |
|------------|----------|
| `TELEGRAM_BOT_TOKEN` | Токен бота Telegram от [@BotFather](https://t.me/BotFather). |
| `ADMIN_TELEGRAM_IDS` | ID администраторов Telegram через запятую (числа). Им доступны управление белым списком и все команды бота. |
| `SITE_URL` | Публичный URL веб-интерфейса (например, `https://fab.example.com`). По нему формируются ссылки «добавить доступ» из бота. |

### Веб-сервер

| Переменная | По умолчанию | Описание |
|------------|--------------|----------|
| `HTTP_PORT` | `8080` | Порт HTTP-сервера. |
| `HOST` | `0.0.0.0` | Интерфейс прослушивания (`0.0.0.0` — все интерфейсы). |

### Безопасность и сессии

| Переменная | По умолчанию | Описание |
|------------|--------------|----------|
| `SECRET_KEY` | генерируется при старте | Секретный ключ Flask (сессии, подписи). В продакшене задать вручную (например, `python -c "import secrets; print(secrets.token_urlsafe(32))"`). |
| `ACCESS_TOKEN_EXPIRY` | `3600` | Время жизни ссылки на веб-доступ в секундах (например, 3600 = 1 час). |

### Прокси и доп. URL

| Переменная | По умолчанию | Описание |
|------------|--------------|----------|
| `TELEGRAM_API_PROXY` | пусто | Адрес прокси для запросов к Telegram API (если пустой — не используется). |
| `VK_API_PROXY` | пусто | Адрес прокси для запросов к VK API (если пустой — не используется). |
| `SITE_BACKUP_URL` | пусто | Дополнительная ссылка для доступа; если задана — выводится с отступом в одну строку под основной ссылкой. Если пустая — не используется. |
| `NGINX_ENABLED` | `false` | `true`/`1`/`yes`, если приложение за nginx: учитывается заголовок `X-Forwarded-For` и т.п. |

### База данных

| Переменная | По умолчанию | Описание |
|------------|--------------|----------|
| `DATABASE_PATH` | `data/fab.db` | Путь к файлу SQLite (БД создаётся при первом запуске). |

### MQTT (опционально)

При `MQTT_ENABLED=false` брокер не используется; при `true` обязательны `MQTT_HOST`, `MQTT_CLIENT_ID`.

| Переменная | По умолчанию | Описание |
|------------|--------------|----------|
| `MQTT_ENABLED` | `false` | Включить публикацию событий в MQTT (например, для MikroTik). |
| `MQTT_HOST` | — | Хост MQTT-брокера (обязателен при `MQTT_ENABLED=true`). |
| `MQTT_PORT` | `1883` | Порт MQTT-брокера. |
| `MQTT_CLIENT_ID` | — | Идентификатор клиента MQTT (обязателен при `MQTT_ENABLED=true`). |
| `MQTT_USERNAME` | пусто | Имя пользователя MQTT (если нужна авторизация). |
| `MQTT_PASSWORD` | пусто | Пароль MQTT. |
| `MQTT_KEEPALIVE` | `60` | Интервал keepalive в секундах. |
| `MQTT_QOS` | `1` | Уровень QoS (0, 1 или 2). |
| `MQTT_TOPIC_PREFIX` | `mikrotik/whitelist/ip` | Префикс топиков для публикации (добавление/удаление IP). |

### Исключаемые сети (всегда открыты)

| Переменная | По умолчанию | Описание |
|------------|--------------|----------|
| `EXCLUDE_IPS` | частные и тестовые CIDR | Список CIDR через запятую. IP из этих сетей не публикуются в MQTT и считаются «всегда открытыми» (например, 10.0.0.0/8, 192.168.0.0/16). |

### Логирование

| Переменная | По умолчанию | Описание |
|------------|--------------|----------|
| `LOG_LEVEL` | `INFO` | Уровень логов: `DEBUG`, `INFO`, `WARNING`, `ERROR`. |

### VK-бот

| Переменная | По умолчанию | Описание |
|------------|--------------|----------|
| `VK_ENABLED` | `false` | Включение VK-бота. По умолчанию выключен; для запуска нужны `VK_ENABLED=true` и `VK_BOT_TOKEN`. |
| `VK_BOT_TOKEN` | — | Токен сообщества/бота ВКонтакте. Если не задан или пустой — VK-бот не запускается, приложение не падает. |
| `ADMIN_VK_IDS` | пусто | ID администраторов VK через запятую (числа). Админы могут управлять белым списком VK. |
| `VK_GROUP_ID` | — | ID группы VK (для Long Poll). Опционально, если можно получить из API по токену. |
